## 0. Hard Boundaries

- 不写正文。
- 不模仿任何特定作者。
- 不复刻原作句子、桥段、人名、设定或关系链。
- 不使用原作人物和具体事件。
- 标题公式 out of scope；本文件只设计 **in-series hook lines**，用于每篇结尾或下一篇开头。
- coded language 只作为平台语感观察，不堆砌，不写灰色操作细节。
- 移民、身份、资产、家庭矛盾只写叙事功能，不写操作性建议。
- 女主必须有主动选择、策略空间、误判和反向控制能力。
- 本项目不能写成单纯“高净值客户八卦合集”；每个 case 都必须推动女主主线。

# 1. Series Positioning

## 1.1 One-line Positioning

> 一个年轻海外安置顾问，在服务高净值候鸟家庭的过程中，逐渐发现太太们真正缺的不是房子、学校和表格，而是能重新获得选择权的女性网络。

## 1.2 Series Type

| 维度 | 定位 |
|---|---|
| 平台形态 | 小红书 / RedNote 口吻的长线连载故事 |
| 类型核心 | 女性网络 + 高净值家庭安置 + 金丝雀文学情绪变体 |
| 结构 | 女主主线 + 单元剧 case |
| 情绪浓度 | 清醒、暧昧、危险、窥视、羡慕、心酸、反向掌控 |
| 主要爽点 | 低位服务者逐渐掌握规则、太太们从被安排到互相托底、男性/公司/合同权力被重新解释 |
| 主要风险 | 写成八卦合集、写成移民服务说明书、写成普通霸总文、爱情线稀释或过度 |

## 1.3 Core Genre Adaptation

本系列不是传统“一个金主 + 一个金丝雀”的单线关系，而是把金丝雀机制拆成一个 **家庭 / 公司 / 合同 / 身份 / 女性网络** 的复合结构：

```text
传统金丝雀机制：
金主提供资源 → 女性被供养 / 展示 / 控制 → 女性觉醒策略

《候鸟太太俱乐部》机制：
先生 / 公司 / 合同提供正式资源 → 太太承担真实生活风险 → 女主掌握隐性需求与信任网络 → 女性网络重新分配信息和选择权
```

## 1.4 Core Creative Question

> 当钱、身份、房子、学校都在合同里，但真正承受生活后果的人不在合同上，她们如何重新拿回一点选择权？

# 2. 核心情绪承诺

## 2.1 Primary Emotional Promise

> 读者每看一篇，都能看到一个体面家庭背后的不安，也看到一个年轻女性如何在别人的不安里建立自己的边界、定价权和判断力。

## 2.2 Emotional Ingredients

| 情绪 | 读者体验 | 剧情来源 |
|---|---|---|
| 窥视感 | 看见高净值家庭海外生活的背面 | 学校、房子、身份、下午茶、家庭安排 |
| 金丝雀感 | 太太们被安置、被展示、被控制，也在积累自己的筹码 | 先生付钱、太太执行、孩子承压、资源不完全归属 |
| 暧昧危险 | 女主与资源方之间有欣赏、利用、试探、亏欠 | 公司老板、本地资源男、付钱先生、强势太太 |
| 女性互助 | 太太们并非纯受害者，她们能互换信息、互相托底 | 候鸟太太俱乐部形成 |
| 清醒爽感 | 女主不恋爱脑，不当救世主，逐渐学会拒绝 | 边界建立、客户筛选、独立定价 |
| 心酸 | 体面生活与真实无助并存 | 豪宅里没有决策权、名校里没有安全感 |
| 反向掌控 | 女主没有钱，却拥有信任、信息模式和连接能力 | 女性网络、方法论、独立客户 |

## 2.3 What Readers Should Keep Thinking

- 她到底是在服务客户，还是在被客户、公司、先生同时消耗？
- 太太们到底是被养着的人，还是家庭风险真正的承担者？
- 女主知道这么多秘密，她会用它们保护谁，又会拒绝谁？
- 一个没有正式权力的人，能不能靠信任和边界变成规则制定者？

# 3. Controlled Emotional Entanglement Layer

Mini Bible Stress Test 的有效机制是女性网络和资源结构，但情感浓度偏职场 / 服务业。Series Bible v0 需要加入 **controlled emotional entanglement layer**：有暧昧、亏欠、试探、嫉妒、保护欲和危险，但不让女主变成恋爱脑。

## 3.1 情感张力人物结构

| 人物关系 | 情感张力 | 服务的权力差 | 控制方式 |
|---|---|---|---|
| 女主 Annie × 公司老板 Alan Shen / 沈总 | 欣赏、提拔、压榨、亏欠 | 他掌握客户池、合同、职位；她掌握客户信任 | 他不能成为霸总男主，只能是职业资源压力源 |
| 女主 Annie × 本地资源男 David / David Lu | 暧昧、互相利用、救场、边界试探 | 他掌握本地资源；她掌握高净值客户入口 | 每次合作必须有利益交换，不写无条件拯救 |
| 女主 Annie × 梁太太 | 依赖、投射、嫉妒、女性亲密 | 梁太太有钱无权；女主无钱有行动力 | 情感亲密必须推动女主边界成长 |
| 女主 Annie × 孟太太 | 压迫式导师感、试探、认可 | 孟太太有判断力和资源，但不轻易信人 | 让女主被迫变强，不让她被收编 |
| 梁太太 × 缺席先生 | 亏欠、控制、旧爱残留 | 先生付钱，太太承担生活 | 不写成单纯渣男，要写远程权力与情感债 |
| 许太太 × 太太网络 | 老江湖式诱惑和危险 | 她知道规则，也能污染规则 | 用她提醒女主：女性网络也可能吃人 |

## 3.2 情感张力如何服务权力差

| 情感张力 | 权力差功能 |
|---|---|
| 欣赏 | 让女主误以为自己被正式承认，后来发现只是被高效使用。 |
| 暧昧 | 让本地资源男的帮助变得不完全免费，女主必须判断代价。 |
| 亏欠 | 让客户太太对女主产生依赖，也让女主难以退出。 |
| 嫉妒 | 太太之间互相比较资源、丈夫、孩子、自由度，暴露隐藏权力结构。 |
| 保护欲 | 资源方看似保护女主或太太，实际可能重新定义她们的位置。 |
| 旧情 | 先生和太太之间并非纯交易，旧情让控制更难切断。 |
| 被需要 | 女主最危险的诱惑不是爱情，而是“只有我能帮她们”。 |

## 3.3 如何避免恋爱脑

女主的情感线必须遵守以下规则：

1. **每次心动都必须伴随风险判断。** 不能只有甜和暧昧。
2. **每次被帮助都要记录代价。** 谁欠谁、欠什么、什么时候会被讨回。
3. **女主不能为男性资源方破坏自己的底线。** 她可以犹豫，但不能长期失去判断。
4. **真正推动女主成长的是边界，不是被爱。**
5. **女主可以被吸引，但不能把被吸引误判成被拯救。**
6. **女性网络的价值不能被 romance 替代。** 爱情张力只是调味层，不是主结构。

## 3.4 如何避免普通霸总文

| 普通霸总文倾向 | 本系列纠偏 |
|---|---|
| 单一强男主解决一切 | 没有单一男主。资源来自公司、本地渠道、太太网络、合同结构。 |
| 男主给钱给房给身份 | 资源方只能提供局部通道，不能替女主完成成长。 |
| 女主被保护就安全 | 被保护也可能意味着被控制、被定义、被收编。 |
| 爱情战胜现实 | 现实结构不会因心动消失，反而会因心动变复杂。 |
| 女主最终靠男人上桌 | 女主靠信任、边界、方法论和客户筛选上桌。 |

## 3.5 Controlled Emotional Entanglement 的使用频率

| 篇章位置 | 情感浓度建议 |
|---|---|
| Episode 1–2 | 轻微：建立女主被需要感、客户依赖感 |
| Episode 3–4 | 中等：老板压力 + 本地资源男救场 + 梁太太私密信任 |
| Episode 5–6 | 加强：太太网络内嫉妒 / 试探，女主被迫表态 |
| Episode 7–8 | 危险：公司压迫、许太太诱导、David 边界试探 |
| Episode 9–10 | 反转：女主拒绝被情感绑架，正式建立规则 |

# 4. 女主完整成长线

## 4.1 女主基础设定

| 项目 | 设定 |
|---|---|
| 姓名 | Annie |
| 年龄 | 27 |
| 城市 | 北美西海岸虚构城市“柏湾”及周边富人区 |
| 职业 | 高净值家庭海外安置服务顾问 |
| 工作范围 | 流程协调、生活安置、学校沟通、客户沟通、应急资源转接；不是律师，不提供法律 / 财务 / 移民专业判断 |
| 背景 | 普通中产家庭出身，留学后留在海外工作，自己也经历过身份焦虑和孤独生活 |
| 外在形象 | 可靠、清爽、嘴严、会照顾人，容易被客户当成“自己人” |
| 核心能力 | 把情绪需求翻译成可执行安排；在多方矛盾信息里判断谁真正处于风险中 |
| 核心缺陷 | 容易过度负责，害怕失去客户，不敢拒绝强势客户和老板 |

## 4.2 初始误判

Annie 一开始误判了三件事：

1. **误判太太们。** 她以为太太们只是情绪化、要求多、消费能力强；后来发现她们是家庭风险最早的感知者。
2. **误判公司。** 她以为公司卖的是专业服务；后来发现公司最想变现的是她和客户之间的信任。
3. **误判自己。** 她以为自己只是执行者；后来发现她掌握的是跨家庭的信息模式、生活系统和真实需求。

## 4.3 欲望

| 层级 | 欲望 |
|---|---|
| 表层欲望 | 做好工作、拿提成、保住身份和职业路径 |
| 中层欲望 | 被客户和行业认可，拥有独立定价权 |
| 深层欲望 | 不再只是替别人安置生活，而是为自己建立一个可持续的位置 |

## 4.4 资源缺口

| 缺口 | 剧情表现 |
|---|---|
| 稳定身份 / 职业安全 | 她不能随便丢工作，也不能轻易得罪公司 |
| 独立客户 | 客户属于公司，不属于她 |
| 行业资历 | 年轻、非律师、非财务顾问，容易被低估 |
| 定价权 | 她做大量情绪劳动，但合同里没有这项价值 |
| 边界 | 客户把她当朋友，公司把她当销售入口，先生把她当执行工具 |
| 资源保护 | 她知道很多秘密，但没有成熟机制保护自己 |

## 4.5 边界建立

Annie 的成长不是“越来越会帮人”，而是“越来越知道什么不能帮”。

| 阶段 | 边界动作 |
|---|---|
| Episode 1–2 | 不把太太私密情绪直接转给先生或公司 |
| Episode 3–4 | 把客户需求写成可执行事项，不写八卦性隐私 |
| Episode 5–6 | 设立太太网络规则：只共享资源，不共享未经授权的隐私 |
| Episode 7–8 | 拒绝公司交出不该交出的信任细节，拒绝太太圈越界请求 |
| Episode 9–10 | 拒绝先生监控式请求，提出独立业务或新规则 |

## 4.6 反向控制能力

Annie 没有钱、没有豪宅、没有正式权力，但她拥有四种反向控制能力：

| 能力 | 说明 |
|---|---|
| 信任入口 | 太太们愿意先告诉她真实情况，而不是告诉公司或先生 |
| 信息模式识别 | 她能看出不同家庭危机背后的共性：预算、身份、学校、控制、孤独 |
| 转译能力 | 她能把“我害怕”“我不敢说”转译成生活安排和风险分流 |
| 退出能力 | 她逐渐学会拒绝高风险客户，不再被每个求助绑定 |

## 4.7 女主成长线总表

| 阶段 | 她以为自己在做什么 | 她实际学到什么 | 成长结果 |
|---|---|---|---|
| 初始 | 服务客户、解决琐事 | 琐事背后是权力结构 | 从执行者变成观察者 |
| 进入太太圈 | 维护关系、提高续费 | 女性网络比公司 CRM 更真实 | 从员工变成信任节点 |
| 第一次越界危机 | 帮客户解决急事 | 帮忙也会制造依赖和责任 | 开始建立边界 |
| 公司压迫 | 保住工作 | 公司想拥有她的信任劳动 | 开始争取定价权 |
| 网络成形 | 帮太太们互助 | 网络也需要规则，否则会互相伤害 | 从连接者变成规则制定者 |
| 反转 | 选择离开或谈判 | 她有了可带走的方法论和客户信任 | 从被选择变成选择客户 |

# 5. 女性网络结构

## 5.1 网络核心逻辑

“候鸟太太俱乐部”不是一个正式组织，而是一个逐渐形成的非正式女性信息网络：

```text
国际学校接送线
→ 下午茶 / 生日会 / 房产看房
→ 私下求助
→ 资源互换
→ 边界冲突
→ 女主建立规则
```

它的核心不是“太太们都很善良”，而是：

- 她们都有资源，但资源不完整。
- 她们都体面，但体面不等于安全。
- 她们会互助，也会试探和利用。
- 她们需要女主，但也可能吞掉女主。

## 5.2 核心太太角色表

### 5.2.1 梁太太：精致陪读太太

| 项目 | 设定 |
|---|---|
| 表面 | 住海景豪宅，孩子上顶级私校，穿着精致，社交能力强 |
| 欲望 | 保住孩子的学校、家庭体面和自己在海外生活中的位置 |
| 秘密 | 家里的预算和资产安排并不由她控制，她只有消费权限，没有决策权限 |
| 资源 | 太太圈入口、学校家长群、房产和生活资源、社交场合 |
| 风险 | 容易把女主当成情绪支柱；也会因为害怕失控而隐瞒关键信息 |
| 与女主关系 | 第一个把女主带进太太圈的人；对女主有依赖、欣赏，也有轻微占有欲 |
| 情感张力 | 她羡慕女主自由，女主羡慕她体面；两人互相误读 |

### 5.2.2 孟太太：强势前高管

| 项目 | 设定 |
|---|---|
| 表面 | 冷静、不合群、说话直接，对服务人员要求极高 |
| 欲望 | 保住自己的判断权，不被丈夫、公司和太太圈 infantilize |
| 秘密 | 她曾经在国内有自己的职业身份，来到海外后被家庭角色吞掉 |
| 资源 | 商业判断、合同意识、风险管理经验、潜在独立客户资源 |
| 风险 | 过度控制，不轻易信任任何人，也可能把女主当成可培养工具 |
| 与女主关系 | 压迫式导师；第一个逼问女主“你到底站哪边”的人 |
| 情感张力 | 女主怕她、讨厌她，又最需要她的清醒提醒 |

### 5.2.3 温太太：新移民妈妈

| 项目 | 设定 |
|---|---|
| 表面 | 温柔、慌张、刚到海外，对系统完全陌生 |
| 欲望 | 让孩子顺利入学，维持家庭稳定，不惹丈夫不高兴 |
| 秘密 | 她没有完全掌握家庭文件和付款信息，很多安排只能通过先生远程确认 |
| 资源 | 读者代入视角、第一批真实信任、对女主的情感依赖 |
| 风险 | 容易过度依赖女主，也容易在压力下把责任推给女主 |
| 与女主关系 | 女主第一个真正想保护的客户 |
| 情感张力 | 不是暧昧，而是“被需要”的危险：女主会想替她扛太多 |

### 5.2.4 许太太：老江湖

| 项目 | 设定 |
|---|---|
| 表面 | 熟悉所有学校、房子、医生、中介和太太圈规则，笑眯眯，消息灵通 |
| 欲望 | 维持自己在太太网络里的信息优势和分配权 |
| 秘密 | 她过去也被别人利用过，所以现在习惯先利用别人 |
| 资源 | 大量本地资源、旧客户、太太圈信用、灰色边界提醒 |
| 风险 | 会试探女主底线，把女主拖进不该进入的家庭隐私 |
| 与女主关系 | 既是资源入口，也是最大污染源 |
| 情感张力 | 她像未来的反面女主：如果 Annie 没有边界，可能变成她 |

### 5.2.5 Ada：女主前辈

| 项目 | 设定 |
|---|---|
| 表面 | 已离开行业，开一家安静的花店 / 小工作室 |
| 欲望 | 不再被客户和公司消耗，但仍关心这个行业里的女性 |
| 秘密 | 当年因为一个家庭 case 越界承担太多，最终被公司和客户同时抛下 |
| 资源 | 经验、警告、旧客户口碑、独立服务模式雏形 |
| 风险 | 她的创伤会让女主看到最坏未来，但她也可能过度悲观 |
| 与女主关系 | 未来警示牌；告诉女主“被需要不是职业护城河” |
| 情感张力 | 代际镜像：女主既想证明自己不会失败，又害怕成为她 |

## 5.3 她们如何互助 / 利用 / 试探

| 行为 | 互助面 | 利用 / 试探面 |
|---|---|---|
| 分享学校信息 | 帮新来的妈妈少踩坑 | 用信息换站队或人情 |
| 推荐医生 / 司机 / 房源 | 提供生活安全感 | 通过资源链条抽取控制权 |
| 下午茶邀请 | 让新人进入社交圈 | 测试对方家庭预算和先生态度 |
| 帮忙接孩子 | 实际互助 | 可能换取隐私和软肋 |
| 私下找女主 | 获得可信赖支持 | 绕过公司和合同，把风险转给女主 |
| 送礼 | 表达感谢 | 绑定人情和未来请求 |

## 5.4 女性网络规则 v0

女主逐渐建立的规则：

1. 只共享公共资源，不共享未经授权的私人信息。
2. 不用孩子作为谈判筹码。
3. 不替任何一方监控另一方。
4. 情绪可以被听见，但决定必须由当事人承担。
5. 所有专业问题转给正式专业人士。
6. 女主可以协调，但不做任何家庭的秘密保管箱。
7. 太太网络不是反先生联盟，而是女性生活安全网。

# 7. 单元剧 Case Template

## 7.1 每个家庭故事必须服务女主主线

每个 case 不能只是“一个有钱家庭的狗血故事”。它必须至少推动以下一项：

- 女主建立新边界。
- 女主理解一种新型权力差。
- 女性网络增加一个规则或裂缝。
- 公司 / 先生 / 本地资源方的压力升级。
- 女主获得或失去某种资源。
- 女主的误判被纠正。

## 7.2 Case Template

```markdown
## Case [编号]: [家庭 / 太太 / 场景代号]

### Surface Problem
表面问题：学校、房子、身份流程、孩子、医疗、搬家、社交、服务投诉等。

### Hidden Conflict
隐藏冲突：预算控制、家庭信息不对称、太太无决策权、先生远程控制、孩子承压、公司销售压力等。

### Canary Mechanism
金丝雀机制：谁被供养、被展示、被安置、被控制、被误判？

### Female Strategy
太太的策略：隐瞒、试探、求助、交换资源、保孩子、保体面、保退路。

### Heroine Strategy
女主的策略动作：协调、拒绝、转介、记录、设边界、建立规则。

### Emotional Entanglement
情感纠缠：谁对谁有亏欠、依赖、暧昧、嫉妒、保护欲？

### Material Detail
物质 / 资源细节：文件、钥匙、账单、房子、学校、礼物、车、群聊、合同等。

### Mainline Impact
对女主主线的推动：边界、定价权、女性网络、公司压力、独立可能性。

### Ending New Variable
结尾新变量：新秘密、新资源、新风险、新人物、新误判。
```

## 7.3 Case 类型库

| Case 类型 | 表面问题 | 隐藏冲突 | 女主主线作用 |
|---|---|---|---|
| 入学危机 | 孩子材料缺失 / 面试失败 | 太太不知道关键文件在哪里 | 女主第一次获得太太私密信任 |
| 豪宅不安全 | 房子漂亮但生活动线混乱 | 太太只有居住权，没有控制权 | 女主理解“住得好不等于安全” |
| 先生远程指令 | 付款方要求女主汇报 | 服务对象和付款方冲突 | 女主建立客户授权边界 |
| 太太圈下午茶 | 社交邀请 | 信息交换和站队试探 | 女性网络成形 |
| 生日会事故 | 孩子社交 / 家长比较 | 家庭体面工程崩裂 | 女主看到孩子承压 |
| 本地资源救场 | 临时房源 / 医疗 / 维修 | 资源男要求回报 | 女主处理暧昧与利益交换 |

| 投诉危机 | 先生投诉服务 | 公司要求女主道歉 | 女主决定谈判或离开 |
| 独立客户 | 太太介绍新客户 | 独立网络出现 | 女主进入新阶段 |

# 13. Proxy-Bias Prevention Checklist

Original Concept Pack v0 暴露过“替身 / 傀儡 / proxy bias”。本系列需持续防止女主或太太们被写成纯工具人。

## 13.1 女主层面

| 风险 | 纠偏 |
|---|---|
| 女主只是替公司安抚客户 | 每 2–3 篇必须出现一次她主动设边界或重构规则。 |
| 女主只是太太们的跑腿 | 每个 case 必须让她获得判断、资源、客户筛选能力或方法论。 |
| 女主被老板 / David 牵着走 | 男性资源方可以提供通道，但女主必须判断代价并保留退出权。 |
| 女主过度救世主 | 她必须拒绝某些请求，承认自己不能救所有人。 |
| 女主只有被需要 | 她最终要从“被需要”转向“被尊重和付费”。 |

## 13.2 太太层面

| 风险 | 纠偏 |
|---|---|
| 太太们只是被养的金丝雀 | 每位太太必须有自己的策略、误判和资源。 |
| 太太们只是八卦素材 | 每个太太 case 必须推动女性网络规则或女主成长。 |
| 太太们都善良互助 | 网络中必须有试探、利用、嫉妒和边界污染。 |
| 太太们都无助 | 她们在某些方面弱，在某些方面比女主强。 |

## 13.3 结构层面

- 每篇必须有女主策略动作。
- 每篇必须有权力差变化。
- 每篇不能只有客户问题，必须有女主主线推进。
- 每 3 篇至少出现一次女主主动拒绝。
- 每 5 篇至少出现一次女主重新定义规则。

# 14. Romance-Dilution Prevention Checklist

Mini Bible Stress Test 的问题是情感纠缠弱化。Series Bible v0 需要加情感浓度，但不能滑向普通恋爱文。

## 14.1 必须保留的情感元素

| 情感元素 | 使用方式 |
|---|---|
| 暧昧 | Annie × David，救场与利益交换之间的微妙感。 |
| 欣赏 / 提拔 | Annie × Alan Shen，职业认可和压榨并存。 |
| 嫉妒 | 太太之间比较自由度、先生态度、孩子资源，而不只是比较包。 |
| 亏欠 | 太太依赖女主，女主也被这种依赖绑住。 |
| 旧情 | 太太与先生不是纯对立，旧情让控制更复杂。 |
| 占有欲 | 梁太太希望女主多帮自己一点，Alan Shen 希望女主把信任交给公司。 |
| 被保护的诱惑 | 资源方提供帮助时，女主要判断这是保护还是收编。 |

## 14.2 每篇 emotional layer 检查

每篇至少具备其中一个：

- 有人对女主表达特别信任。
- 有人让女主产生亏欠。
- 有人用欣赏包装控制。
- 有人因为女主帮了别人而嫉妒。
- 有人试图保护女主，但代价不透明。
- 女主对某人的判断发生情感性动摇。
- 女主选择边界，放弃短期情感回报。

## 14.3 防止 romance 过度稀释

| 稀释风险 | 纠偏 |
|---|---|
| 只有客户 case，没有暧昧 / 心痛 / 亏欠 | 每篇加入 controlled emotional tension 栏。 |
| 女主过于冷静，像职业工具人 | 允许她心软、被打动、犹豫，但最终要做策略判断。 |
| David 线太工具化 | 让他每次帮助都有不透明代价，也有真实欣赏。 |
| Alan Shen 线太职场化 | 加入“认可成瘾”：女主曾经很想被他认可，后来脱敏。 |
| 太太们太像客户 | 增加依赖、嫉妒、投射、亏欠、占有欲。 |

## 14.4 防止变成普通霸总文

- 男性不能解决核心问题，只能制造或缓解局部压力。
- 女主不能因为心动放弃客户边界。
- 任何暧昧都必须引入资源代价。
- 最终让女主上桌的是规则和网络，不是被某个男人爱上。

# 15. 是否适合进入 3-post MVP

## 15.1 结论

**适合，而且建议进入 3-post MVP。**

原因：

1. **平台理解成本低。** 海外生活、陪读、太太圈、学校、房子、家庭矛盾都是 RedNote 读者容易理解的素材。
2. **女性网络强。** 能显著纠正“女主只是替身 / 工具人”的 proxy bias。
3. **金丝雀情绪可以被恢复。** 通过 controlled emotional entanglement，可以补上暧昧、亏欠、嫉妒、情感危险。
4. **单元剧可持续。** 每个家庭 case 都能引入新变量，但必须扣回女主主线。
5. **安全边界可控。** 只写叙事功能，不写操作性建议。

## 15.2 MVP 3-post 选择

建议 MVP 使用 Episode 1–3，但每篇都要明确测试目标。

| MVP Post | 测试目标 | 必须出现 |
|---|---|---|
| Post 1 | 顾问视角能否建立高净值家庭秘密感 | 入学危机、温太太私密请求、女主被需要的危险 |
| Post 2 | 太太圈是否有平台传播力 | 下午茶、太太互相试探、女性网络入口 |
| Post 3 | 女主主线是否成立 | 公司销售压迫、Alan Shen 认可与控制、女主第一次设边界 |

## 15.3 MVP 成功标准

| 指标 | 成功信号 |
|---|---|
| 读者是否追女主 | 评论关注“Annie 会怎么选”，而不只是问客户八卦 |
| 太太圈是否成立 | 读者能区分梁太太、温太太、孟太太 / 对太太网络感兴趣 |
| 情绪浓度是否足够 | 读者感到“体面但危险”“被需要但窒息”“清醒但心酸” |
| 金丝雀感是否迁移成功 | 读者能感受到供养、安置、展示、控制、选择权缺失 |
| 安全边界是否清楚 | 内容没有变成移民 / 资产 / 家庭纠纷操作指南 |

## 15.4 MVP 前最终 Guardrails

生成 Post 1–3 前，应固定以下规则：

- 不写现实可执行的移民、资产、监控、家庭纠纷操作建议。
- 不使用真实机构名、学校名、公司名或可识别地点。
- 每篇必须有女主策略动作。
- 每篇必须有一个物质 / 资源细节推动剧情。
- 每篇必须有 controlled emotional tension。
- 每篇结尾必须有 new variable + in-series hook line。
- 女主不能被写成单纯跑腿、救世主或客户情绪垃圾桶。
